仔细观察身边事物的运行方式，千差万别的东西可能被抽象为相似的模型。 仔细观察身边事物的运行方式，相似的外表下可能是极其不同的原理；相似的抽象模型与抽象性不那么显然的事实结合，可能导致完全不同的东西。 成像技术的发展，使得信息的动态展现十分方便。数字电路技术的发展，使得计算的机械化十分方便。电子信息技术于是发展起来，信息时代到来。 显示功能的需求在提高，通讯功能的需求在提高，数据相关功能的需求在提高；就像一个文本编辑器满足不了所有人一样，编程语言层出不穷，各自面向的核心需求不尽相同，实现不同功能的方便程度也不同。 尽管编程语言众多，它们都能被抽象到逻辑的计算这一模型。PL 的研究，算法的研究是它们共同的地方。 然而我们应该看到区别，不同语言之间的关系，可以像画家与作家，建筑与音乐那样，各自最关心最直接最要紧的不是抽象设计，而是实际功能。 建筑与音乐不仅是凝固与否的区别，性质不同的应用适合的选用语言也会不同。 就像方便 view 设计的 js，方便 server 设计的 python，php，方便 data 分析的 R。绕了半天，其实这才是我最初想到的。
